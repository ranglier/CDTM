# Roadmap

## Court terme

- finaliser l'administration technique epuree
- brancher l'editeur cartographique sur les tables separees `localities`, `landmarks`, `forces`
- enrichir les champs dynamiques de case

## Contraintes structurelles

- pas de notes dans la carte
- pas de `visibilite`
- pas de tri manuel `sort_order`
- pas de retour au modele `map_points` generique

## Prochaines briques

- edition des objets cartographiques sur la carte
- publication selective `draft / published / archived`
- outillage de migration et d'import pour les objets de carte
